# 相机动态识别后续问题

记录日期：2026-07-10
状态：待后续修复

本轮只修复扫描结果展示与多帧共识。以下问题保留为独立后续工作，避免在同一次改动中同时调整相机参数、质量阈值和卡片兼容策略。

## 2. 缺少相机帧质量门控

### 现状

- `AndroidOmrImageAnalyzer` 会测量 `laplacianVariance`，但不据此拒绝模糊帧。
- `DeviceStabilityMonitor` 只显示“请持稳手机”，不控制帧是否进入 OMR。
- 强过曝和较强模糊可能产生 `success=true` 的错误分数，不能假设坏帧一定会识别失败。

### 后续方案

- 新增纯 JVM 的 `FrameQualityEvaluator`，只在卡片区域评估清晰度、曝光裁剪比例、局部对比度、卡片面积和定位重投影误差。
- 手机连续稳定约 300ms 后，在 400～600ms 候选帧中选择质量最高的一帧进入完整 OMR。
- 阈值必须由真机日志标定，不能直接恢复未经标定的固定 Laplacian 阈值。

### 验收标准

- 模糊、严重过曝和欠曝帧被明确拒绝，不产生错误成功结果。
- 正常室内光照下不会因门槛过严导致扫描长期无响应。
- 增加录制相机帧序列的回归测试，并覆盖曝光变化、运动模糊和失焦。

## 3. CameraX 捕获与采样配置未闭环

### 现状

- 分析请求为 1280×960，但允许回退到更低分辨率；旧 L 角标探针在 960 宽时无法定位。
- 分析器每 300ms 处理一帧，实际约 3.3fps，没有在候选帧中择优。
- 固定请求 `[30,30]` AE FPS，但没有根据设备能力选择，也没有读取 AF/AE 收敛状态。
- Preview 与 ImageAnalysis 没有通过统一 `ViewPort` 绑定，取景与分析区域可能不完全一致。

### 后续方案

- 检查并记录实际分析分辨率；宽度低于 1280 时拒绝扫描或切换更合适的配置。
- 性能允许时评估 1600/1920 宽分析流，并用真实设备测量耗时和热量。
- 配置连续自动对焦和卡片区域测光；仅采纳 AF/AE 已收敛的候选帧。
- 从 Camera2 能力列表选择支持的 FPS range，并使用统一 `UseCaseGroup`/`ViewPort`。
- 保留 `STRATEGY_KEEP_ONLY_LATEST`，将“每 300ms 随机取帧”改为“低成本逐帧评估、择优送入 OMR”。

### 验收标准

- 调试信息包含实际分辨率、AF/AE 状态、曝光时间、ISO、帧时间戳和丢帧原因。
- 预览引导框与分析帧中的卡片区域一致。
- 至少在两台不同相机能力的 Android 设备上验证配置兼容性。

## 4. 旧卡片角标仍有静默错分风险

### 现状

- 新模板默认使用编码角标，1280 宽的轻度模糊/噪声回归结果稳定。
- legacy L 角标在强过曝时仍可能在 `outer` 与 `innerRight` 参考之间切换，已复现 10/10 变成 8/10 的静默错分。
- 已打印的旧卡仍必须保留兼容路径，不能直接删除。

### 后续方案

- 新打印卡继续使用编码角标，并在产品提示中逐步引导旧卡换印。
- legacy L 路径出现参考候选分歧时降级为可见失败，不输出分数。
- 为旧卡增加强过曝、阴影、旋转和运动模糊的专门回归集。

### 验收标准

- legacy L 路径宁可返回未识别，也不得在已知强过曝样例上输出错误成功分数。
- 编码角标、实心角标和 L 角标三条兼容路径都有独立的真图回归测试。
