# 编码角标答题卡模板实施计划

## 目标

将新生成答题卡的四个相同实心方块升级为四个可区分位置与方向的编码角标，降低答案涂块、画面偏移和单角遮挡导致的误定位。保持 `omr-core` 为纯 JVM，保留旧实心方块和 L 形角标扫描兼容。

## 模板格式

- 每个角标沿用现有 `CORNER_MARKER_SIZE` 和中心位置，避免改变题目、学号区以及旧模板的整体几何。
- 角标使用 6×6 网格：最外一圈固定为黑色定位边框，内部 4×4 为编码区。
- LU、RU、LD、RD 使用四个不同码字；任意码字及其 90° 旋转之间保持至少 6 bit 汉明距离。
- 角标四周不连接连续黑框，保留白色隔离区，避免与标题、答案格和纸边连成同一组件。
- `TemplateRenderer` 和桌面测试渲染器默认输出编码角标；旧 `SOLID_SQUARE`、`L_BRACKET` 仅用于兼容和回归。

## 识别链路

1. 全局阈值与连通域找近方形黑色外环候选。
2. 从候选边界提取四个极值角点，建立候选局部透视映射。
3. 采样 6×6 单元，校验黑色边框，并对 4×4 编码区做旋转和汉明距离解码。
4. 按角标 ID 建立模板角点到相机角点的多点对应；检测到至少三个不同 ID 时，用不少于 12 个点拟合全卡 homography，并补出缺失角标中心。
5. 四个中心转换为现有 `MiniProgramAnchors`，下游继续复用投影、读格和评分逻辑。
6. 新模板优先走编码角标；完全没有编码角标证据时才依次回退旧实心方块和 L 形角标。若只解出 1～2 个编码角标则可见失败，避免把受损新卡误送旧路径。

## 可靠性约束

- 解码最多允许 2 bit 错误，且最优码与次优码必须有明确距离差。
- 至少识别三个不同位置的角标；同一 ID 的多个候选选择解码错误数和边框质量最优者。
- 对拟合后的所有角点计算重投影误差，误差过大时拒绝，不将低置信度几何送入读格。
- 新模板角标路径使用唯一来源标识，确保下游采用角标中心参考，而不是旧 L 形外角参考。
- 对缺角结果保留可观测 debug 信息：识别 ID、解码旋转、bit 错误数、重投影误差和补点数量。

## 验证范围

本机没有 Android SDK，因此本轮验证限定为：

- `omr-core` 几何和码字单测；
- 纯 JVM 桌面模板像素测试；
- 平面、透视、模糊、噪声以及遮挡一个角标的完整扫描测试；
- 旧实心方块和 L 形卡回归。

Android 机器后续执行：

- `sh gradlew test`
- `sh gradlew assembleDebug`
- 导出 PNG 后检查打印尺寸、角标白区和 1280×960 CameraX 实拍识别。

## 后续阶段

本轮不加入分带辅助点和纸张弯曲模型。编码角标稳定后，可在每个 15 题分带左右增加辅助标记，并将单一 homography 升级为分带 homography。
